# 参与贡献

感谢你帮助改进 CampusCrawler。

## 开发流程

1. Fork 仓库并创建功能分支。
2. 保持改动范围清晰，不提交抓取结果、个人信息或网站凭据。
3. 为新增解析规则补充测试。
4. 运行 `python -m unittest discover -s tests -v`。
5. 提交 Pull Request，说明目标、测试方法和涉及的网站类型。

## 功能边界

我们接受公开网页解析、结果整理、导出和可访问性方面的改进。

以下功能不会被接受：

- 绕过登录、验证码、付费墙、限速或访问控制
- 账号、Cookie、Token 或凭据共享
- 代理池、验证码识别或隐蔽规避检测
- 未授权访问私有接口、数据库、个人信息或非公开文件
- 可能影响目标网站正常运行的高频或大规模采集

对网站授权或数据公开状态存在疑问时，请先开 Issue 讨论，不要直接提交实现。
